# p0-w3-d4-t4 · Безопасный вывод: textContent против innerHTML

> Тип: объёмная (важная тема дня: XSS) · Репозиторий: один отдельный

> Подключить `.js` к HTML через `<script>`, отсмотреть в браузере.

## Задача

Есть два способа вставить содержимое в элемент — `textContent` и `innerHTML` — и разница между ними про безопасность. Пользовательский ввод, вставленный через `innerHTML`, может протащить на страницу чужой HTML и даже скрипт (XSS). Разбираешь это на живом примере — ленте сообщений.

```
вывод пользовательского текста — двумя способами
  textContent = ввод → безопасно (теги как текст)
  innerHTML  = ввод → ОПАСНО (теги/скрипт сработают)

строка-ловушка: <img src=x onerror=...>
лента сообщений: добавляем безопасно
```

## Дано (готовая верстка)

```html
<input id="msg" placeholder="сообщение (попробуй с тегами)">
<button id="sendText">Отправить (textContent)</button>
<button id="sendHtml">Отправить (innerHTML)</button>
<ul id="feed"></ul>
```

Тестовые вводы: `<b>привет</b>` и строка-ловушка `<img src=x onerror="alert('xss')">`.

## Что нужно сделать (по релизам)

**Release 1 — два вывода.** По «Отправить (textContent)» добавлять новое сообщение в ленту, вставляя текст через `textContent`. По «Отправить (innerHTML)» — то же, но через `innerHTML`. Ввести обе тестовые строки обоими способами и сравнить.

**Release 2 — разбор.** В `NOTES.md` описать, что показал каждый способ на `<b>привет</b>` и на строке-ловушке, и почему `innerHTML` с пользовательским вводом опасен.

**Release 3 — правильная лента.** Оставить добавление сообщений только безопасным способом (`textContent`): создавать `<li>` через `createElement`, класть текст через `textContent`, добавлять в `#feed`. Показать, что теперь строка-ловушка выводится как безобидный текст.

## Ожидаемый результат

Через `textContent` теги видны как текст (`<b>привет</b>` со скобками), ловушка безвредна. Через `innerHTML` теги срабатывают (жирный текст), а ловушка пытается выполнить код. Итоговая лента использует только безопасный вывод.

## Критерии приёмки

* Показана разница `textContent` и `innerHTML` на одном вводе, включая строку-ловушку.
* В `NOTES.md` объяснено, почему пользовательский ввод выводят через `textContent` и когда `innerHTML` допустим (только свой доверенный HTML).
* Итоговая лента добавляет сообщения безопасно (`createElement` + `textContent`).
